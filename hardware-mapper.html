⬡ Software Architecture — Universal hardware mapper design

ARCHON — How This Software Works

A 7-layer architecture that detects, maps, and visualises the complete internal hardware topology of any device — including real-time electron flow and metal/alloy identification when external probes are connected.

L1

Platform Bootstrap

Detects OS, kernel version, privilege level. Selects correct API backend: WMI (Windows), sysfs/procfs (Linux), IOKit (macOS), Build.HARDWARE (Android).

L2

Hardware Enumeration

CPUID instruction → ISA, topology, features. SMBIOS Type 0–45 → board, BIOS, serial. lspci/WMI → full PCIe device tree. lsusb → USB hub tree. NVMe IDENTIFY + SMART.

L3

Sensor & Telemetry Layer

RAPL (per-core power), MSR reads (voltage/turbo), LPCIO/Super-I/O (fan RPM, Vcore, temps), ACPI battery state, SMC on macOS, EDID from displays.

L4

Signal Flow Inference

From topology + performance counters: models signal direction CPU→PCIe→GPU/NVMe, CPU→IMC→DIMMs, USB host→hubs→devices. Bandwidth utilisation per link.

L5

External Probe Bridge

USB-C HID/CDC protocol to resistivity scanner, eddy-current probe, magnetic current imager, logic analyser. Streams 100 readings/sec into alloy-ID engine.

L6

Electron Flow Renderer

WebGL particle system. Electrons animated along bus lanes proportional to real bandwidth. Signal direction, bottlenecks highlighted. Clock pulses visualised.

L7

3D Reconstruction

SMBIOS + die database + probe resistivity map + thermal overlay → 3D spatial model of PCB and components, showing metal composition per trace.
